# 致所有读者 (To Everyone)

欢迎阅读本书！我们希望你在阅读时的享受，能如我们写作时一样多。本书名为《操作系统：三大易懂篇章》（*Operating Systems: Three Easy Pieces*，可在 [http://www.ostep.org](http://www.ostep.org) 获取）。书名显然是在向理查德·费曼（Richard Feynman）在物理学领域创作的史上最伟大的讲义之一致敬 [F96]。虽然本书无疑达不到那位著名物理学家所设下的高标准，但对于想要探索操作系统（乃至更广泛的计算机系统）本质的你来说，或许已经足够用了。

这“三个易懂篇章”指的是本书组织架构的三个主要主题元素：**虚拟化 (Virtualization)**、**并发 (Concurrency)** 和 **持久性 (Persistence)**。在探讨这些概念的过程中，我们最终会涉及操作系统所做的大部分重要工作；希望你在这段旅程中也能体会到乐趣。学习新知识是件乐事，对吧？至少，它本该如此。

每个主要概念都被划分为一系列章节，其中大多数章节都会先提出一个特定的问题，然后展示如何解决它。这些章节都很短小精悍，并尽可能地引用了这些思想最初的源材料。我们写这本书的目标之一，就是让历史的脉络尽可能清晰，因为我们认为这有助于学生更清晰地理解“现在是什么”、“过去是什么”以及“未来会怎样”。在这种情况下，看看香肠是如何制作出来的，几乎和弄清楚香肠有什么用一样重要[^1].

[^1]:提示：用来吃的！或者如果你是素食主义者，那就离它远点。

在整本书中，我们使用了一些特定的叙述手法，大概值得在这里介绍一下。首先是**问题的关键 (Crux of the problem)**。每当我们试图解决一个问题时，我们首先会尝试陈述最核心的问题是什么；这种“问题的关键”会在正文中被明确指出，并希望通过后续提出的技术、算法和思想来加以解决。

在许多地方，我们将通过展示系统随时间推移的行为来解释它的工作原理。这些**时间线 (Timelines)** 是理解的本质；举个例子，如果你知道当一个进程发生**缺页中断 (Page Fault)** 时会发生什么，你就已经迈向了真正理解**虚拟内存 (Virtual Memory)** 运作方式的道路。如果你能理解当**日志文件系统 (Journaling File System)** 将一个数据块写入磁盘时发生了什么，你就已经迈出了精通存储系统的第一步。

正文中还穿插了大量的**补充说明 (Asides)** 和**提示 (Tips)**，为主线内容增添了一抹色彩。补充说明往往讨论一些与主文相关（但可能并非绝对必要）的内容；提示则往往是能应用于你所构建系统中的通用经验教训。为了方便起见，书末的索引列出了所有的提示和补充说明（以及 cruces，也就是 crux 奇怪的复数形式）。

我们在整本书中还使用了最古老的教学方法之一——**对话 (Dialogue)**，以此来从不同的角度呈现一些材料。这些对话被用来引入主要的主题概念（正如我们将看到的那样，以一种非常棒的方式），以及时不时地复习一下材料。这也是一个用更幽默的风格进行写作的机会。至于你觉得它们是否有用，或者是否幽默，嗯，那就是另一回事了。

在每个主要部分的开头，我们会首先提出一个由操作系统提供的**抽象 (Abstraction)**，然后在随后的章节中探讨提供该抽象所需的**机制 (Mechanisms)**、**策略 (Policies)** 以及其他支持。抽象是计算机科学各个方面的基础，因此它们在操作系统中同样至关重要，这或许并不令人意外。

在各个章节中，我们尽量使用**真实代码 (Real Code)**（而不是伪代码），因此对于几乎所有的示例，你应该都能自己把它们敲出来并运行。在真实的系统上运行真实的代码是学习操作系统的最佳方式，所以我们鼓励你在可能的情况下尽量这么做。为了方便大家查阅，我们也将代码发布在了 [https://github.com/remzi-arpacidusseau/ostep-code](https://github.com/remzi-arpacidusseau/ostep-code)。

在正文的各个部分，我们穿插了一些**课后作业 (Homeworks)**，以确保你理解了正在发生的事情。许多作业是操作系统中各个小部件的微型模拟器；你应该下载这些作业，并运行它们来测验自己。这些作业模拟器有这样一个特点：通过赋予它们不同的随机种子 (Random Seed)，你可以生成几乎无限的问题集；模拟器还可以被设定为你解答这些问题。因此，你可以不断地测试和重新测试自己，直到达到一个很好的理解水平。

本书最重要的补充内容是一系列**项目 (Projects)**，在这些项目中，你将通过设计、实现和测试自己的代码来了解真实系统是如何工作的。所有的项目（以及前面提到的代码示例）都是用 **C 语言** 编写的 [KR88]；C 语言是一门简单而强大的语言，构成了大多数操作系统的底层基础，因此非常值得将其加入你的编程语言工具箱。目前有两种类型的项目可供选择（有关想法，请参阅在线附录）。第一种是**系统编程项目 (Systems Programming Projects)**；这些项目非常适合那些刚接触 C 语言和 UNIX，并希望学习如何进行底层 C 语言编程的人。第二种是基于麻省理工学院 (MIT) 开发的一个名为 **xv6** 的真实操作系统内核的项目 [CK+08]；这些项目非常适合那些已经有一定 C 语言基础，并且想要亲自动手深入操作系统内部的学生。在威斯康星大学，我们以三种不同的方式开设了这门课程：要么全做系统编程项目，要么全做 xv6 编程项目，或者两者混合。

我们正在慢慢公开项目描述和一个测试框架。更多信息请参见 [https://github.com/remzi-arpacidusseau/ostep-projects](https://github.com/remzi-arpacidusseau/ostep-projects)。如果你没有在学校选修这门课，这将为你提供一个独立完成这些项目的机会，以便更好地学习相关材料。不幸的是，当你卡壳时，你没有助教 (TA) 可以去烦，但生活中并非一切都是免费的（不过书可以是！）。
